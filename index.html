Video Autopause Demo

Two strategies for pausing video when users switch away — and why they behave differently.

window blur / focus

Fires when the browser window loses or regains OS focus.
Triggers on: alt-tab to another app, clicking DevTools, opening a native dialog.
Does NOT fire on tab switch within same window.

Page Visibility API

Fires when the tab becomes hidden or visible.
Triggers on: switching tabs, minimizing window, screen lock.
Does NOT fire when window focus moves to another app while tab is visible.
window.blur / focus

Window Focus Detection

Pauses when the OS window loses focus — even if this tab is still visible.

Video: waiting
Window: —
window.addEventListener("blur", () => video.pause());
window.addEventListener("focus", () => video.play());

Tab Visibility Detection

Pauses when the tab becomes hidden — regardless of which app has OS focus.

Video: waiting
Tab: —
document.addEventListener("visibilitychange", () => {
  if (document.hidden) video.pause();
  else video.play();
});
